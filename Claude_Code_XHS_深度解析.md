## 第1页：Agentic Loop 核心循环机制

### 图示：流式智能体循环

```
┌─────────────────────────────────────────────────────────┐
│                    User Input                           │
│              "帮我重构用户认证模块"                        │
└─────────────────────────┬───────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│              1. 上下文加载（并行）                        │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐    │
│  │Git 分支  │ │最近提交  │ │CLAUDE.md │ │工作目录  │    │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘    │
└─────────────────────────┬───────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│              2. LLM 推理（流式输出）                     │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 正在分析代码结构... [████████░░░░░] 67%          │    │
│  │ 发现了 3 个需要重构的函数...                     │    │
│  └─────────────────────────────────────────────────┘    │
└─────────────────────────┬───────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│              3. 工具执行（LLM 还在生成时就开始）          │
│  Tool: Grep     │  Tool: Glob   │  Tool: Read          │
│  ┌───────────┐ │ ┌───────────┐ │ ┌───────────┐        │
│  │ 搜索函数   │ │ │ 查找文件  │ │ │ 读取代码  │        │
│  └───────────┘ │ └───────────┘ │ └───────────┘        │
└─────────────────────────┬───────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│              4. 结果聚合 → 继续推理 → 直到任务完成       │
└─────────────────────────────────────────────────────────┘
```

### 技术要点

- **流式并行执行**：LLM 还在生成输出时就开始执行工具调用，不是等完全生成再执行
- **每轮重新加载上下文**：每次查询都会重新读取 Git 信息、CLAUDE.md 等
- **提示缓存复用**：稳定的提示边界重用 KV Cache，节省 token 成本
- **结构化搜索**：使用 Grep/Glob 而非原始 Shell，获得更结构化的结果

## 第2页：多智能体架构（三层设计）

### 图示：三层智能体架构

```
                    ┌─────────────────┐
                    │   团队（Team）   │
                    │  用户顶层交互    │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │   协调器层       │
                    │  (Orchestrator)  │
                    │  一个 Claude     │
                    └────────┬────────┘
                             │ 邮箱系统通信
          ┌──────────────────┼──────────────────┐
          │                  │                  │
┌─────────▼─────────┐ ┌──────▼──────┐ ┌───────▼──────┐
│    子智能体 A      │ │   子智能体 B  │ │   子智能体 C   │
│  (代码探索者)      │ │  (代码架构师) │ │  (测试工程师)  │
│                   │ │              │ │              │
│  Git Worktree A  │ │ Worktree B  │ │ Worktree C   │
└───────────────────┘ └─────────────┘ └──────────────┘

         隔离的 Git Worktree（防止相互污染）
```

### 技术要点

- **三层结构**：子智能体 → 协调器 → 团队
- **协调器模式**：一个 Claude 实例管理多个并行工作的子智能体
- **工作区隔离**：子智能体在独立的 Git Worktree 中运行，不会相互冲突
- **零成本并行**：子智能体使用 KV Cache 分叉，并行化几乎零额外 token 成本
- **应用场景**：代码审查（code-review）、功能开发（feature-dev）等专业工作流

## 第3页：工具系统设计（40+ 工具）

### 图示：工具调用架构

```
┌──────────────────────────────────────────────────────────┐
│                    自然语言指令                           │
│         "帮我把这个函数重构一下，加上错误处理"             │
└─────────────────────────┬────────────────────────────────┘
                          ▼
┌──────────────────────────────────────────────────────────┐
│                   意图识别 & 路由                          │
└─────────────────────────┬────────────────────────────────┘
                          ▼
        ┌─────────────────┼─────────────────┐
        ▼                 ▼                 ▼
┌───────────────┐ ┌───────────────┐ ┌───────────────┐
│   BashTool    │ │  FileWriteTool │ │    GrepTool   │
│               │ │               │ │               │
│ ⚠️ 沙箱化执行  │ │  差异计算优化  │ │  结构化搜索   │
│ 禁止 Unix Socket │ │  支持大文件   │ │  支持正则    │
└───────────────┘ └───────────────┘ └───────────────┘
        │                 │                 │
        └─────────────────┼─────────────────┘
                          ▼
              ┌───────────────────────┐
              │      LSP 集成        │
              │  语言服务器协议       │
              │  - 符号定义          │
              │  - 调用层次          │
              │  - 引用查找          │
              └───────────────────────┘
```

### 技术要点

- **BashTool 沙箱化**：Linux 下使用 `apply-seccomp` 阻止访问 Unix 套接字
- **PowerShell 加固**：包含对后台任务绕过、TOCTOU 漏洞的检查
- **FileWriteTool 优化**：差异计算优化，处理大文件时速度显著提升
- **LSP 集成**：直接利用语言服务器协议获取代码结构信息
- **Grep/Glob 结构化**：专用工具替代原始 Shell，获得更精确的结果

## 第4页：三层记忆架构

### 图示：记忆系统

```
┌─────────────────────────────────────────────────────────┐
│                   三层记忆架构                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  第1层：MEMORY.md（主索引）                              │
│  ┌─────────────────────────────────────────────────┐    │
│  │ ## 项目上下文                                     │    │
│  │ ## 技术栈                                        │    │
│  │ ## 最近的改动                                    │    │
│  │ ## 待完成的任务                                  │    │
│  └─────────────────────────────────────────────────┘    │
│                         │                               │
│                         ▼                               │
│  第2层：按需加载的主题文件                               │
│  ┌────────────┐ ┌────────────┐ ┌────────────┐          │
│  │ auth.md    │ │ api.md     │ │ db.md      │          │
│  │ 认证详情    │ │ API 文档   │ │ 数据库设计  │          │
│  └────────────┘ └────────────┘ └────────────┘          │
│                         │                               │
│                         ▼                               │
│  第3层：完整会话记录（可搜索）                            │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 今天 14:32 - 用户要求重构认证模块                  │   │
│  │ 今天 14:35 - 分析了 12 个相关文件                  │   │
│  │ 今天 14:40 - 完成了 3 个函数的重构                │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 技术要点

- **严格的写入纪律**：只有确认文件写入成功后才更新索引，防止失败尝试污染上下文
- **验证后再行动**：智能体将记忆视为"提示"，行动前根据实际代码验证
- **按需加载**：不是所有记忆都加载，按需读取避免上下文溢出
- **会话恢复**：强大的会话管理，中断后可继续工作

## 第5页：五层权限系统

### 图示：权限控制模型

```
┌─────────────────────────────────────────────────────────┐
│                   五层权限系统                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  第1层：系统级（管理员配置）                              │
│  ┌─────────────────────────────────────────────────┐   │
│  │ • SOC 2 Type 2、ISO 27001 认证                   │   │
│  │ • 企业级强制策略，故障关闭                        │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  第2层：项目级（.mcp.json 配置）                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │ • 白名单 MCP 服务器                              │   │
│  │ • 允许的命令列表                                │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  第3层：用户级（~/.claude.json）                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │ • 个人偏好设置                                   │   │
│  │ • API 密钥管理                                  │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  第4层：会话级（运行时权限）                             │
│  ┌─────────────────────────────────────────────────┐   │
│  │ • 默认只读，修改需授权                           │   │
│  │ • 命令黑名单                                     │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  第5层：工具级（每个工具独立权限）                        │
│  ┌─────────────────────────────────────────────────┐   │
│  │ BashTool: 可执行 / 禁止网络 / 沙箱范围            │   │
│  │ FileWrite: 可写目录 / 禁止目录                   │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 技术要点

- **基于每个工具的权限模型**：限制智能体操作出错时的爆炸半径
- **沙箱隔离**：文件系统 + 网络双重隔离
- **命令黑名单**：危险命令默认禁止
- **故障关闭**：无法获取托管设置时阻止启动
- **提示注入检测**：即使被恶意操控，沙箱确保系统安全

## 总结

Claude Code 的架构设计体现了几个关键工程决策：

1. **流式优先**：不是等 LLM 完全生成再执行，而是流式并行
2. **记忆持久化**：三层记忆架构解决跨会话遗忘问题
3. **安全第一**：五层权限 + 沙箱隔离，可信赖的 AI 执行
4. **多智能体协作**：三层架构支持复杂任务的分解执行
5. **工程化设计**：不是 AI+工具的简单组合，而是完整的智能体工程系统
