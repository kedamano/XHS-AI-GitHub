# 小红书 AI 博主技能

## 概述

这是一个专门用于将 GitHub 上的 AI 项目转化为小红书爆款种草内容的工作流技能。帮助用户生成具有强吸引力、高转化率的图文内容，适用于 AI 工具推荐、项目分享、科技种草等场景。

## 触发条件

当用户说以下内容时使用此技能：
- "帮我写一篇小红书"
- "生成 AI 项目种草内容"
- "推荐一个 GitHub AI 项目"
- "写一个 AI 工具分享笔记"
- 任何涉及 GitHub AI 项目 + 小红书内容的请求

## 核心工作流

### 第一步：项目选择标准

从 GitHub 趋势项目中筛选 AI 项目时，遵循以下优先级：

**必须满足：**
1. 项目必须与 AI 相关（GPT、Agent、AIGC、自动化工具、LLM 应用等）
2. 项目对普通用户有实际价值（拒绝纯技术底层库、SDK、框架）
3. 项目有可操作的界面或 API（CLI 工具需有图形化替代方案）

**加分项：**
- 近期 star 增长迅速（24h/7d 内）
- 有 Demo 或在线体验地址
- 支持中文或有多语言界面
- 被知名开发者或媒体推荐

### 第二步：爆款标题生成

生成 5 个备选标题，遵循以下公式：

**情绪型公式：**
- "救命！这个 AI 工具也太好用了吧😭"
- "姐妹们！这个让我效率翻倍的 AI 神器必须知道"
- "用了 3 个月，彻底离不开这个 GitHub 爆款 AI 项目"

**数字型公式：**
- "5 分钟搞定 XX，这个 GitHub 10k+ star 的 AI 项目绝了"
- "实测 7 款 AI 写作工具，这款是 yyds"

**对比型公式：**
- "不吹不黑，这个 GitHub AI 项目比某付费软件还好用"
- "从月薪 3 千到 3 万，全靠这个 AI 工具"

**FOMO 型公式：**
- "全网都在找的 AI 工具，我给你们扒到了"
- "这个 GitHub 项目不用，你绝对会后悔"

**禁忌：**
- 避免纯技术描述（"这是一个基于 Transformer 的..."）
- 避免平淡陈述（"给大家介绍一个项目"）
- 避免过长标题（超过 25 字）

### 第三步：封面文案

生成 1 句封面文案，要求：

**特征：**
- 短（5-12 字）
- 有视觉冲击力
- 引发好奇或共鸣

**模板：**
- "效率翻倍的神仙 AI ⚡️"
- "用了再也回不去😭"
- "这个 AI 让我偷懒成功了"
- "GitHub 爆款！必须收藏"
- "免费 AI 工具 yyds"

### 第四步：正文结构

按以下顺序生成完整正文：

```
【开头钩子】(必选)
- 提出痛点或场景
- 用口语化表达引发共鸣
- 例子："你们有没有这种感觉..."

【项目介绍】(必选)
- 口语化介绍项目功能
- 不要堆砌技术术语
- 用类比帮助理解

【核心功能】(必选，3-5 条)
- 用 emoji 标记每个功能
- 简短说明 + 实际应用场景
- 避免技术细节

【推荐理由】(必选)
- 为什么选择这个而不是其他
- 适合什么样的用户

【适合人群】(必选)
- 明确目标读者
- 用"你"来称呼

【真实感评价】(必选)
- 加入个人使用体验
- 可以加入"踩坑提醒"
- 语气要像真人博主

【互动引导】(必选)
- 引导评论区互动
- 例子："评论区告诉我你们想看什么"
```

### 第五步：热门标签

生成 10-15 个标签，策略如下：

**核心标签（必选）：**
- #AI #人工智能 #AI工具
- #GitHub #开源
- #效率工具 #学习必备

**领域标签（根据项目选择）：**
- #ChatGPT #AI写作 #AI绘画
- #AI助手 #AI效率 #AI办公
- #AI副业 #AI赚钱 #学生党
- #打工人 #新媒体 #设计师

**趋势标签：**
- #2025AI #AI新物种 #AIGC
- #黑科技 #神器推荐

### 第六步：风格要求

**语气特征：**
- 像真人博主，不要像 AI
- 多用短句，一句话一行
- 适当 emoji，但不过度
- 情绪要有，但不过度夸张

**禁忌：**
- 不要写"作为一个 AI 语言模型"
- 不要用"首先、其次、最后"的僵硬结构
- 不要堆砌官方介绍
- 不要写太长的段落

**加分技巧：**
- 加入"我实际用了"的语气
- 可以加入踩坑提醒
- 制造 FOMO（错过可惜的感觉）

## 质量检查清单

生成内容后，自查以下要点：

- [ ] 标题是否足够吸引人（有情绪/数字/对比）
- [ ] 封面文案是否短而有冲击力
- [ ] 开头钩子是否能引发继续阅读的欲望
- [ ] 正文是否口语化（朗读是否顺口）
- [ ] 功能介绍是否对普通人有价值
- [ ] 是否有个人体验/评价（真实感）
- [ ] 是否有互动引导
- [ ] 标签是否覆盖核心关键词
- [ ] 是否有 FOMO 元素
- [ ] 是否避免了技术术语堆砌
